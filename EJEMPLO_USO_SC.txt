# Ejemplos de uso del script con el archivo SC.txt

## Opción 1: Usar el archivo SC.txt con todos los subscriber codes

# Prueba pequeña (10 usuarios)
python test_rendimiento_completo.py --host http://192.168.4.204:8000 --users 10 --subscriber-codes-file SC.txt

# Prueba media (100 usuarios)
python test_rendimiento_completo.py --host http://192.168.4.204:8000 --users 100 --subscriber-codes-file SC.txt --max-concurrent 20

# Prueba completa (1000 usuarios)
python test_rendimiento_completo.py --host http://192.168.4.204:8000 --users 1000 --subscriber-codes-file SC.txt --max-concurrent 50 --timeout 60

## Opción 2: Usar un subscriber code individual

python test_rendimiento_completo.py --host http://192.168.4.204:8000 --users 100 --subscriber-code 1

## Opción 3: Combinar archivo + subscriber code individual

python test_rendimiento_completo.py --host http://192.168.4.204:8000 --users 100 --subscriber-codes-file SC.txt --subscriber-code EXTRA123

## Notas importantes:

1. El script distribuirá los subscriber codes de forma aleatoria entre los usuarios
2. Si tienes 60 subscriber codes en SC.txt y simulas 1000 usuarios, cada código se usará múltiples veces
3. El archivo SC.txt debe estar en la misma carpeta que el script, o proporciona la ruta completa
